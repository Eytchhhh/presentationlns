WHAT IS API?

An API is a set of programming code that enables data transmission between one software product and another. It also contains the terms of this data exchange.

WHAT IS REST/ful API?

A REST API (also known as RESTful API) is an application programming interface (API or web API) that conforms to the constraints of REST architectural style and allows for interaction with RESTful web services. REST stands for representational state transfer and was created by computer scientist Roy Fielding.

To be termed RESTful, an API must meet the following requirements:

  • Clients, servers, and resources make comprise a client-server architecture, with HTTP requests managing requests.

  • Client-server communication that is stateless means that no client information is stored between get requests and that each request is distinct and unrelated.

  • Data that can be cached to speed up client-server interactions.

ANATOMY OF API REQUEST

ENDPOINT

The endpoint (or route) is the url you request for.

METHOD

The method is the type of request you send to the server.These methods provide meaning for the request you’re making. They are used to perform four possible actions: Create, Read, Update and Delete (CRUD). You can choose from these five types below:

  • GET
  • POST
  • PUT
  • PATCH
  • DELETE

DATA OR BODY

The data (sometimes called “body” or “message”) contains information you want to be sent to the server. This option is only used with POST, PUT, PATCH or DELETE requests.

HEADERS

Headers are used to provide information to both the client and server. It can be used for many purposes, such as authentication and providing information about the body content.

HOW DOES IT WORKS?

The key elements of the REST API paradigm are:
  • a client or software that runs on a user’s computer or smartphone and initiates communication;
  • a server that offers an API as a means of access to its data or features;
  • a resource, which is any piece of content that the server can provide to the client (for example, a video or a text file).

BEST PRACTICES IN CREATING A REST API/API

If you want to make your API user's life easier and smoother, you should stick to the best REST API design principles to prevent tripping over your API's syntactic mess. The following are tried- and-true conventions to follow when creating REST APIs:
